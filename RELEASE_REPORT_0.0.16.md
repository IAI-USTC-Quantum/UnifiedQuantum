### B-2 ‚Äî `uv sync` universal resolution fails (pre-existing, documented)
- Severity: Medium (documented limitation; user path unaffected)
- Affected area: developer install
- Evidence: `uv sync --extra all --group dev --group docs` and even `uv sync --group dev` fail: "there are no versions of `quarkcircuit{python_full_version >= '3.12' and < '3.14' and sys_platform != 'win32'}`". No `uv.lock` exists. `pyproject.toml` is unchanged from v0.0.15.
- Reproduction: `uv sync --extra all --group dev --group docs` on a fresh clone.
- Expected: documented install command resolves.
- Actual: universal lock fails because the `[quark]` extra (kept out of `[all]`) is still evaluated by uv's universal resolver and is unsatisfiable for 3.12‚Äì3.14.
- Suggested owner/fix: maintainer ‚Äî either (a) keep documenting "use `uv pip install`, avoid `uv sync`" (already in `docs/source/1_basic_usage/best_practices.md`), (b) add a committed `uv.lock`, or (c) tighten the `quark` extra marker so it does not trigger cross-version resolution. Not a 0.0.16 regression; does not block tagging on its own.
